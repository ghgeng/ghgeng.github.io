# 2.配置Vue

提示

Vue中的el是id选择器，选择了叫demo的容器，这样容器和vue就建立了链接，也可以用class选择器

想让Vue工作，就必须创建一个Vue实例，且要传入一个配置对象；

<script type="text/javascript" >
    // Vue.config.productionTip = false //阻止 vue 在启动时生成生产提示。
    //创建Vue实例
    new Vue({
        el:'#demo' //el用于指定当前Vue实例为哪个容器服务，值通常为css选择器字符串。
    })

</script>

# 3.配置中的Data

用对象的形式放入想要的值，通过vue的插值语法{{}}两个大括号赋值，vue自动填充数据，

<script type="text/javascript" >
    // Vue.config.productionTip = false //阻止 vue 在启动时生成生产提示。

    //创建Vue实例
    new Vue({
        el:'#demo', //el用于指定当前Vue实例为哪个容器服务，值通常为css选择器字符串。
        data:{ //data中用于存储数据，数据供el所指定的容器去使用，值我们暂时先写成一个对象。
            name:'atguigu',
            address:'北京'
        }
    })
</script>

# 问题二.

  • 由下图看出，第二个容器vue并没有解析，其实vue只用第一个容器，下边的容器不再用了，反之也是这样，两个vue实例接管同一个容器，虽然被接管了，但是不生效
    • 一个vue实例不能同时接管两个容器,Vue实例和容器是一一对应的
    • 真实开发中只有一个Vue实例，并且会配合着组件一起使用

# 问题三.

  • 容器只有一个，实例只有一个，如果有一百个数据，需要一百个key-value嘛，这就涉及到组件化，可以让这一百个展示的数据交给vue组件，这就把数据拆开了

# 问题五.

  • 怎么看到实时数据更新?

# 总结

1.想让Vue工作，就必须创建一个Vue实例，且要传入一个配置对象；
2.root容器里的代码依然符合html规范，只不过混入了一些特殊的Vue语法；
3.root容器里的代码被称为【Vue模板】；
4.Vue实例和容器是一一对应的；
5.真实开发中只有一个Vue实例，并且会配合着组件一起使用；
6.{{xxx}}中的xxx要写js表达式，且xxx可以自动读取到data中的所有属性；
7.一旦data中的数据发生改变，那么页面中用到该数据的地方也会自动更新；

注意区分：js表达式 和 js代码(语句)
        1.表达式：一个表达式会产生一个值，可以放在任何一个需要值的地方：
                    (1). a
                    (2). a+b
                    (3). demo(1) 函数调用表达式
                    (4). x === y ? 'a' : 'b'

        2.js代码(语句)
                    (1). if(){}
                    (2). for(){}
