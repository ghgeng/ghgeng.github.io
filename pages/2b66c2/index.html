# 范围值说明

  • compile（编译作用域）: 默认值，表示依赖在所有的classpath中可用，且会被打包，这是最常见的依赖配置。

  • provided（已提供作用域）:
    类似于compile，但是期望JDK或者运行时的容器会提供该依赖。不会被打包。

  • runtime（运行时作用域）:
    表示依赖不是编译时必须，但是运行时需要，会被打包。

  • test（测试作用域）:
    仅在运行测试时使用，不会被打包。

  • system（系统作用域）:
    与provided类似，但必须通过systemPath显式指定依赖文件位置。由于有潜在的依赖性和复杂性，Maven不推荐使用此作用域，除非绝对必要。
    注意：系统作用域的依赖项不会从Maven仓库中解析，并且其路径必须明确指定。

  • import（导入作用域）:
    仅用于<dependencyManagement>部分。它允许你导入另一个POM的<dependencyManagement>部分，并有效地继承其依赖项配置。
    示例：在多模块项目中，父POM可以定义一组依赖项和版本，然后通过import作用域被子模块继承。

# phase

在 Maven 中，构建生命周期（Build Lifecycle）是由一系列的阶段（Phases）组成的，这些阶段定义了构建过程中的各个步骤。Maven 提供了三个主要的生命周期：default（核心）、clean 和 site。每个生命周期都包含了一系列的阶段（Phases），你可以通过 Maven 命令行或 Maven 插件来触发这些阶段。

# 1. Default 生命周期

Default 生命周期是 Maven 的核心生命周期，它包含了构建和部署应用所需的大部分阶段。
以下是 Default 生命周期中常用的阶段（按执行顺序排列）：

  • ources：处理源代码，例如，过滤任何值。
  • generate-resources：生成所有需要包含在打包过程中的资源文件。
  • process-resources：复制并处理资源文件至目标目录，准备打包。
  • compile：编译项目的源代码。
  • process-classes：处理编译后生成的文件，例如，Java 类的字节码增强。
  • generate-test-sources：生成所有需要包含在测试编译过程中的源代码。
  • process-test-sources：处理测试源代码，例如，过滤任何值。
  • generate-test-resources：生成所有需要包含在测试打包过程中的资源文件。
  • process-test-resources：复制并处理测试资源文件至目标测试目录。
  • test-compile：编译测试源代码至目标测试目录。
  • process-test-classes：处理测试编译后生成的文件。
  • test：使用适当的单元测试框架运行测试。
  • prepare-package：在实际打包之前执行必要的操作。
  • package：将编译后的代码打包成可分发的格式，如 JAR、WAR 或 EAR。
  • pre-integration-test：执行集成测试前的操作，如设置所需的环境。
  • integration-test：处理并部署软件包到集成测试环境。
  • post-integration-test：执行集成测试后的操作，如清理集成测试环境。
  • verify：运行所有检查以验证包是否有效且满足质量标准。
  • install：将包安装到本地仓库，以便在本地其他项目中作为依赖使用。
  • deploy：将最终的包复制到远程仓库，以便共享给其他开发者和项目。

# 2. Clean 生命周期

Clean 生命周期包含以下阶段：

  • pre-clean：执行清理前需要完成的工作。
  • clean：删除上一次构建生成的所有文件。
  • post-clean：执行清理后需要完成的工作。

# 3. Site 生命周期

Site 生命周期用于生成项目的站点文档，包括报告和文档等。它包含以下阶段：

  • pre-site：生成项目站点之前需要执行的工作。
  • site：生成项目的站点文档。
  • post-site：生成项目站点之后需要执行的工作。
  • site-deploy：将生成的站点文档部署到特定的服务器上。
