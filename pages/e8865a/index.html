深入理解Java浅拷贝与深拷贝

# 浅拷贝深拷贝的理解

  • 简单说拷贝就是将一个类中的属性拷贝到另一个中，对于BeanUtils.copyProperties来说，你必须保证属性名和类型是相同的，因为它是根据get和set方法来赋值的。
    • 浅拷贝
    • 浅拷贝可以理解为如果是引用类型，那么目标对象拷贝的只是源对象的地址，无论目标对象还是源对象改变，他们都会一起改变，但是浅拷贝只是调用子对象的set方法，并没有将所有属性拷贝,比如引用类型名称相同，但是类型不同，可能导致类型转换错误
    • 如果是基本类型，通过set方法赋值
  • 深拷贝
    • 深拷贝就是将源对象的属性全部复制一份给目标对象，复制完之后他们就是隔开的，没有任何关系，无论操作源对象还是目标对象都对另一个没有影响
  • 无论是浅拷贝还是深拷贝，对于基本类型和String来说都是没有影响的，有影响的只有引用类型数据

# BeanUtils.copyProperties 深拷贝，还是浅拷贝?

  • 是浅拷贝

  • 有子对象就一定不能用BeanUtils么

    • 并不绝对，这个要区分考虑：
    • 1、子对象还要改动。
    • 2、子对象不怎么改动。
      虽然有子对象，但是子对象并不怎么改动，那么用BeanUtils也是没问题的。

# 浅拷贝

  • 上面说的用工具类都是浅拷贝
  • 浅拷贝是按位拷贝对象，它会创建一个新对象，这个对象有着原始对象属性值的一份精确拷贝。如果属性是基本类型，拷贝的就是基本类型的值；如果属性是内存地址（引用类型），拷贝的就是内存地址 ，因此如果其中一个对象改变了这个地址，就会影响到另一个对象。

# 浅拷贝实现方法

  • 1、使用工具类，内容引用改变后，值都会发生改变
 public static void main(String[] args) {

# 深拷贝

  • 深拷贝复制变量值，对于引用数据，则递归至基本类型后，再复制。深拷贝后的对象与原来的对象是完全隔离的，互不影响，对一个对象的修改并不会影响另一个对象。深拷贝相比于浅拷贝速度较慢并且花销较大。

# 深拷贝实现方法 Cloneable

  • (1) 通过重写clone方法来实现深拷贝(实现Cloneable接口)
    • 推荐】慎用 Object 的 clone 方法来拷贝对象。
    • 说明： 对象的 clone 方法默认是浅拷贝，若想实现深拷贝需要重写 clone 方法实现属性对象的拷贝。
  • 与通过重写clone方法实现浅拷贝的基本思路一样，只需要为对象图的每一层的每一个对象都实现Clonneable接口并重写clone方法，最后在最顶层的类的重写的clone方法中调用所有的clone方法即可实现深拷贝。简单的说就是“每一层的每个对象都进行浅拷贝=深拷贝。

Benz方法

package code.Java基础.拷贝.Benz;

import lombok.Data;

import java.io.Serializable;
import java.util.ArrayList;
import java.util.Collections;
import java.util.List;
import java.util.stream.Collectors;

@Data
public class Benz implements Cloneable {

    private String name ;

    private List<Part> list;

    @Override
    public Benz clone() throws CloneNotSupportedException {
        Benz clone = (Benz)super.clone();
        List<Part> list = new ArrayList<>();

        for (int i = 0; i < clone.getList().size(); i++) {
            Part clone1 = (Part) clone.getList().get(i).clone();
            list .add(  clone1 );
        }
        clone.setList(list);
        return clone;
    }
}

# 深拷贝实现方法 Serializable

  • (2)推荐: 通过对象序列化实现深拷贝(实现Serializable接口)
    • 虽然层次调用clone方法可以实现深拷贝，但是显然代码量实在太大。特别对于属性数量比较多、层次比较深的类而言，每个类都要重写clone方法太过繁琐。
    • 将对象序列化为字节序列后，默认会将该对象的整个对象图进行序列化，再通过反序列即可完美地实现深拷贝。
 // 深拷贝

        List<Part> list1 = new ArrayList<>();
        list1.add(part);
        list1.add(part1);

        benz.setList(list1);

        System.out.println("奔驰零部件" + benz.toString());

        Benz benz1 = Test.clone(benz);

        // 修改奔驰零部件
        benz1.setName("奔驰Cl");
        benz.getList().get(0).setPname("宝马发动机");

  // 深拷贝工具类
    public static <T extends Serializable> T clone(T obj) {
        T cloneOjg = null;
        try {
            // 序列化
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream os = new ObjectOutputStream(bos);
            os.writeObject(obj);
            os.close();

            // 反序列化
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream is = new ObjectInputStream(bis);
            cloneOjg = (T) is.readObject();
        } catch (Exception e) {
            e.printStackTrace();
        }
        return cloneOjg;
    }

2、Jackson、fastjson序列化方式

  • Jackson可以将对象序列化成JSON，需要有默认的无参构造函数。
   public static void main(String[] args) throws Exception {
        Benz benz = new Benz();
        benz.setName("奔驰");
        // 零部件
        Part part = new Part();
        part.setPname("发动机");
        Part part1 = new Part();
        part1.setPname("内燃机");

        List<Part> list1 = new ArrayList<>();
        list1.add(part);
        list1.add(part1);

        benz.setList(list1);

        System.out.println("奔驰零部件" + benz.toString());
        System.out.println("使用Jackson序列化实现深拷贝=============");
        // 使用Jackson序列化进行深拷贝
        ObjectMapper objectMapper = new ObjectMapper();
        // 装成json字符串
        String str = objectMapper.writeValueAsString(benz);
        BMW bmw = objectMapper.readValue(str, BMW.class);
        bmw.getList().forEach(System.out::println);

        System.out.println("使用fastjson序列化实现深拷贝===========");
        String string = JSONArray.toJSONString(benz);
        BMW bmw1 = JSON.parseObject(string, BMW.class);
        bmw1.getList().forEach(System.out::println);

        BMW clone = JSONObject.parseObject(JSONObject.toJSONBytes(benz), BMW.class);
        clone.getList().forEach(System.out::println);
    }
