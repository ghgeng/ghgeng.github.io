窗口函数

# 窗口函数

1、在所有的SQL处理中,窗口函数都是最后一步执行,而且仅位于Order by字句之前。
2、当同一个select查询中存在多个窗口函数时,他们相互之间是没有影响的。
3、如果只使用partition by子句,未指定order by的话,我们的聚合是分组内的聚合。
4、order by子句会让输入的数据强制排序。

  • 常用的4类窗口函数：
    • 用于聚合计算的窗口函数：sum() over(); count() over(); avg() over;
    • 用于分组排序的窗口函数：row_number() over(); rank() over(); dense_rank() over()
    • 用于分组查询的窗口函数：ntile() over()
    • 用于偏移分析的窗口函数：lag() over();lead() over();first_value() last_value()

# 1、sum() over()函数

  • 分数表 
select *,sum(score) over(partition by student_name order by id desc) as su from score;
1
  • 在每个student_name内（通过student_name进行分组），根据id顺序，对score进行向下累加,得到结果:最后一个计算的值是前面的总和。
  • 比如张三 id = 11 分数为75

# 2、count() over()函数

select * ,count(student_name) over(partition by student_name order by id  desc)  as su from score;
1
  • 计算函数由sum改成了count，由向下累加改成了向下计数。类似于row_number()，低频使用。
  • avg() over() 函数同理；省略

# 3、row_numer() over() 函数

row_number()的作用是对数据进行排序。

  • 3.1 在student_name分区里面，根据id对每个数据进行排序。
select * ,ROW_NUMBER() over(partition by student_name order by id desc )as rowNum  from score
1
  • 3.2 在student_name、score分区里面，根据id,对每个数据进行排序。
select * ,ROW_NUMBER() over(partition by student_name | score order by id desc )as rowNum  from score
1
  • 3.3 在student_name分区里面，根据score查询分数最高的一门
select * from (
	select * ,ROW_NUMBER() over(partition by student_name order by score desc )as rowNum  from score 
	) aa 
where rowNum = 1;
